# Reproduce

Every command below was run and verified against a current checkout of
this repository before being written down here. All commands are run
from the `costplus_suite/` directory (not the repo root, and not this
`dataset/` folder).

```
git clone <this repository>
cd cpd/costplus_suite
pip install -r requirements.txt
```

Python 3.10+ is assumed (see `costplus_suite/requirements.txt` for pinned
package versions).

## 1. Regenerate the main leaderboard, state breakdown, and state summary

```
python run.py
```

This is the entire pipeline entry point. It live-fetches the current
Cost Plus catalog, CMS NADAC, CMS Medicare Part D Spending by Drug, and
CMS/Medicaid SDUD (see `PROVENANCE.md`, Section A, for exactly how each
is discovered and fetched), crosswalks every drug through RxNorm, joins
`data/public_spreads_matched.csv`'s hand-curated citations (see below --
this file is *not* regenerated by `run.py`), and writes:

- `output/leaderboard.csv`
- `output/leaderboard_by_state.csv`
- `output/state_summary.csv`
- (Module E is enabled by default and additionally writes
  `output/spread_changes.csv`, `output/trumprx_comparison.csv`, and
  `output/brand_price_increase_leaderboard.csv` -- not part of this
  deposit, which only packages the six files listed in
  `dataset/README.md`.)

**Verified output** (run 2026-07-16, live data): `leaderboard.csv`
2,062 rows / 56 confirmed markup rows; `leaderboard_by_state.csv`
70,253 rows; `state_summary.csv` 52 rows. Console summary:

```
Medicare Part D: $8,022,647,897.99
Medicaid: $5.3B
Total: $13.3B
```

Because NADAC refreshes weekly and CMS Part D/SDUD refresh on their own
schedules (see `PROVENANCE.md`, "Change notice"), a re-run on a later
date will not reproduce these exact dollar figures bit-for-bit, but
should reproduce the same drug set and the same order of magnitude.

## 2. Regenerate the catalog-gap list

`output/catalog_gaps.csv` is not written by `run.py` -- it is a separate,
on-demand module that cross-references `data/public_spreads.csv`'s
citations against the live Cost Plus catalog:

```
python -c "
import sys
sys.path.insert(0, '.')
from modules import h_catalog_gaps
h_catalog_gaps.run().to_csv('output/catalog_gaps.csv', index=False)
"
```

**Verified output**: 7 rows (Daptomycin, Enoxaparin Sodium, Fondaparinux
Sodium, Octreotide Acetate, Pregabalin, Sofosbuvir/Velpatasvir,
Vigabatrin).

## 3. Regenerate the unpriced-drug list

`output/unpriced_drugs.csv` is likewise a separate on-demand module. It
scans the *entire* Cost Plus catalog (not just cited drugs) for drugs
that resolve to a real RxCUI with real NDCs but for which NADAC currently
has no price for any of those NDCs:

```
python -c "
import sys, time
sys.path.insert(0, '.')
from modules import i_unpriced_drugs
out = i_unpriced_drugs.run()
out.to_csv('output/unpriced_drugs.csv', index=False)
"
```

**Verified output**: 204 rows / 191 distinct RxCUIs. Takes roughly
15 seconds with a warm NADAC cache (`run.py` populates the cache; run
step 1 first for a fast result here, or expect a longer first run while
NADAC is fetched fresh).

## 4. `public_spreads_matched.csv` -- NOT regenerated by any command

This file is hand/rule-curated, one-time citation extraction: PDF text
from `costplus_suite/data/sources/`, two federal court complaints, and
one research blog post were read (see `PROVENANCE.md`, Section B), and
every drug-level markup figure that satisfied this project's strict
disambiguation rules (documented in `METHODOLOGY.md`, "Public citation
enrichment") was manually entered into
`costplus_suite/data/public_spreads_matched.csv`. There is no script
that reproduces this file from the source PDFs -- reproducing it would
mean re-doing the extraction and re-applying the same manual judgment
calls, not running code. `run.py`'s Module G (`g_public_citations.py`)
only *reads* this file and joins it onto the leaderboard; it never writes
it. Anyone auditing a specific figure should verify it directly against
the `source_name` / `source_page` / `source_quote` columns in that file
(and its copy in this `dataset/` folder) rather than expecting to
re-derive it from a command.

python -m pytest tests/ -q

**Verified**: 172 passed (2026-07-16, same checkout used for step 1).

## 6. The Excel workbook (`output/PBM_Markup_Analysis.xlsx`)

This convenience workbook (4 sheets: PBM Markup, States, Methodology,
Sources) is not part of this data deposit -- the CSVs in this `dataset/`
folder are the canonical machine-readable records; the xlsx is a
human-browsable rendering of `leaderboard.csv` and `state_summary.csv`
built with `openpyxl`, kept in the parent repo's `output/` folder but not
copied into this deposit. There is currently no committed script in the
repository that regenerates it from a clean clone -- it is maintained by
hand alongside the CSVs. This is noted here rather than glossed over: if
a future contributor wants the xlsx to be a first-class reproducible
artifact, it should be turned into a real script under `costplus_suite/`
(analogous to `h_catalog_gaps.py` and `i_unpriced_drugs.py` above) rather
than remaining an ad hoc build step.

## Summary of what is and isn't reproducible by command

| File (in this `dataset/` folder) | Regenerated by | Deterministic? |
|---|---|---|
| `leaderboard.csv` | `python run.py` | No -- drifts with NADAC/Part D/SDUD refreshes |
| `leaderboard_by_state.csv` | `python run.py` | No -- same reason |
| `state_summary.csv` | `python run.py` | No -- same reason |
| `catalog_gaps.csv` | `modules/h_catalog_gaps.py` (command above) | No -- depends on live catalog + `public_spreads.csv` |
| `unpriced_drugs.csv` | `modules/i_unpriced_drugs.py` (command above) | No -- depends on live catalog + NADAC |
| `public_spreads_matched.csv` | Not regenerated -- static, hand-curated | N/A |
| `data_dictionary.csv` (repo root) | Not regenerated -- hand-authored, describes the above | N/A |
